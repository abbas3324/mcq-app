## API Endpoints

- GET `/api/health` — health check
- POST `/api/questions` — accept pasted text and a `category`, parses questions and inserts them into SQLite
- GET `/api/questions` — list all stored questions (optional `?category=`)
- GET `/api/categories` — list distinct categories

## Database

- SQLite DB file: `backend/data.db`
- Table: `questions(id INTEGER PRIMARY KEY, question TEXT, type TEXT, correct TEXT, options TEXT)`

## How to run (development)

1. Backend (from repository root):

```bash
cd backend
npm install
# run via npx to avoid PS execution policy issues
npx ts-node-dev --respawn --transpile-only src/index.ts
```

2. Frontend:

```bash
cd frontend
npm install
npm run dev
```

Open http://localhost:5173/ and paste questions into the form, pick a category and click "Submit Questions" — the app will POST to the backend and start the exam.

## Notes & Troubleshooting

- If port 4000 is already in use, the backend can be started on another port by setting the `PORT` env var, e.g. `set PORT=4001` (Windows) before running the server.
- If TypeScript compilation reports a `moduleResolution` deprecation, `backend/tsconfig.json` includes `"ignoreDeprecations": "6.0"` to suppress the error for the current toolchain.
